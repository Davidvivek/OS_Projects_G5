## Project Overview

This project implements **seven custom UNIX utilities** and a **custom shell** entirely from scratch in C, using low-level system calls (`open`, `read`, `write`, `close`, `fork`, `execvp`, etc.) and POSIX APIs. Each utility mirrors a standard UNIX command with a `custom_` prefix to avoid conflicts.

All commands can be run standalone **or** from within the included `g5_shell` interactive shell.

## Custom Shell — g5_shell

A lightweight interactive shell that reads user commands, parses arguments, and executes them.

**Features:**
- Custom prompt displaying `G5_Shell> `
- Tokenizes user input and passes arguments to child processes
- Uses `fork()` + `execvp()` for command execution
- Parent process waits for child completion using `wait()`
- Graceful exit with the `exit` command

**Logic:**
- Reads a line from `stdin` using `fgets()`
- Strips the trailing newline with `strcspn()`
- Splits the input into tokens using `strtok()` with space as a delimiter
- Forks a child process; the child calls `execvp()` to replace itself with the requested program
- If `execvp()` fails, prints `"Command Not Found!!"`

**Usage:**
```
$ ./bin/g5_shell
G5_Shell> ./bin/custom_ls
G5_Shell> ./bin/custom_cat sample_cat.txt
G5_Shell> exit
```

## Utility Reference

### 1. custom_cat

Concatenates and displays file content to standard output.

**Features:**
- Reads and outputs file content using low-level I/O (`open`, `read`, `write`)
- Efficient 4 KB buffered reading
- Proper error handling for file-open and write failures

**Logic:**
- Opens the file in read-only mode using `open()` with `O_RDONLY`
- Reads data in chunks of 4096 bytes into a buffer using `read()`
- Writes each chunk to `stdout` (file descriptor 1) using `write()`
- Validates that all bytes were written successfully
- Closes the file descriptor on completion

**Usage:**
```
custom_cat <filename>
```
```bash
./bin/custom_cat sample_cat.txt
```

### 2. custom_ls

Lists directory contents with color-coded output.

**Features:**
- Color-coded output: **Cyan** for directories, **Yellow** for device files, **Green** for executables
- Long listing format (`-l`) showing file type, inode, and size
- Summary statistics (`-s`) displaying total directories, files, and bytes
- Skips hidden files (entries starting with `.`)
- Handles both file and directory arguments
- Uses `lstat()` to avoid following symlinks

**Logic:**
- Parses command-line flags (`-l`, `-s`) from `argv`
- If the target is a regular file, prints its info directly
- If the target is a directory, opens it with `opendir()` and iterates entries via `readdir()`
- Applies ANSI color codes based on file type (`S_ISDIR`, `S_ISCHR`, `S_ISBLK`) and permissions (`S_IXUSR`)
- In long mode, pads filenames to a fixed width and prints type, inode, and size
- Accumulates counters for the optional summary report

**Usage:**
```
custom_ls [OPTIONS] [PATH...]
```
| Flag | Description |
|------|-------------|
| `-l` | Long listing format (type, inode, size) |
| `-s` | Print summary (total dirs, files, bytes) |

```bash
./bin/custom_ls
./bin/custom_ls -l /home
./bin/custom_ls -ls /tmp
```

### 3. custom_grep

Searches for a pattern in a file and prints matching lines.

**Features:**
- Case-insensitive search (`-i`)
- Line number display (`-n`)
- Count of matching lines only (`-c`)
- Reads from `stdin` if no file is specified
- Custom `contains()` function for substring matching (no regex dependency)
- Warns about extra arguments that are ignored

**Logic:**
- Parses flags (`-i`, `-n`, `-c`) by scanning option characters
- Copies the pattern; converts to lowercase if `-i` is set using a custom `to_lower()` function
- Reads the file (or `stdin`) line-by-line using `fgets()`
- For each line, creates a comparison copy (lowered if `-i`) and checks for a match using a manual substring search (`contains()`)
- Outputs matching lines with optional line numbers; or outputs just the count if `-c` is set
- Prints `"No match found."` when the pattern is absent

**Usage:**
```
custom_grep [OPTIONS] "PATTERN" [FILE]
```
| Flag | Description |
|------|-------------|
| `-i` | Case-insensitive matching |
| `-n` | Show line numbers |
| `-c` | Print count of matching lines only |

```bash
./bin/custom_grep "hello" sample_grep.txt
./bin/custom_grep -i "HELLO" sample_grep.txt
./bin/custom_grep -n "pattern" sample_grep.txt
./bin/custom_grep -c "error" logfile.txt
```

### 4. custom_wc

Counts lines, words, and characters in one or more files.

**Features:**
- Per-file breakdown of lines, words, and characters
- Aggregated totals when processing multiple files
- Handles whitespace-separated word detection using `isspace()`

**Logic:**
- Iterates over each file argument and opens it with `fopen()`
- Reads character-by-character using `fgetc()`
- Increments the character count on every character
- Increments the line count on every `\n`
- Tracks word boundaries: a word starts when a non-space character follows a space (or start of file), using an `in_word` flag
- Accumulates totals across all files and prints a `TOTAL` summary when 2+ files are given

**Usage:**
```
custom_wc <file1> [file2 ...]
```
```bash
./bin/custom_wc sample_cat.txt
./bin/custom_wc sample_cat.txt sample_grep.txt
```

### 5. custom_cp

Copies a source file to a destination file.

**Features:**
- Efficient 4 KB buffered copy using low-level I/O
- Creates the destination file if it does not exist (permissions `0644`)
- Overwrites the destination if it already exists
- Cross-platform readiness with `O_BINARY` compatibility

**Logic:**
- Opens the source file read-only using `open()` with `O_RDONLY`
- Opens/creates the destination file using `open()` with `O_WRONLY | O_CREAT | O_TRUNC`
- Reads from source in 4096-byte chunks using `read()` and writes each chunk to the destination with `write()`
- Validates that `bytes_written == bytes_read` after every write to detect partial writes
- Checks `bytes_read < 0` after the loop to catch late read errors
- Closes both file descriptors on completion

**Usage:**
```
custom_cp <source_file> <destination_file>
```
```bash
./bin/custom_cp report.txt backup_report.txt
```

---

### 6. custom_mv

Moves or renames a file. Falls back to copy-and-delete across filesystems.

**Features:**
- Fast metadata-only `rename()` when source and destination are on the same filesystem
- Automatic fallback to copy-and-delete when `rename()` fails (e.g., cross-device move)
- Informs the user when the fallback path is taken
- Reuses the same 4 KB buffered copy logic as `custom_cp`

**Logic:**
- First attempts `rename(source, dest)` — a fast, atomic operation when both paths are on the same filesystem
- If `rename()` fails (typically `EXDEV` — cross-device link), falls back to `copy_and_delete()`:
  1. Opens source read-only, creates/truncates destination
  2. Copies data in 4 KB chunks
  3. Validates every write
  4. Deletes the original with `unlink()`
- Prints a notice to `stderr` when the fallback is used

**Usage:**
```
custom_mv <source_file> <destination_path>
```
```bash
./bin/custom_mv old_name.txt new_name.txt
./bin/custom_mv report.txt /tmp/report.txt
```

### 7. custom_rm

Removes files or directories with optional recursive deletion.

**Features:**
- Recursive directory removal (`-r` / `-R`)
- Verbose mode (`-v`) — prints each deletion
- Summary report (`-s`) — shows total files and directories removed
- Safety check: refuses to delete `/`, `.`, or `..`
- Uses `lstat()` to handle symlinks correctly (removes the link, not the target)

**Logic:**
- Parses flags (`-r`, `-v`, `-s`) from command-line arguments
- For each target path:
  - Calls `lstat()` to determine type without following symlinks
  - **If it's a file/symlink:** removes with `unlink()`
  - **If it's a directory and `-r` is set:** opens with `opendir()`, recursively processes all entries (skipping `.` and `..`), then removes the empty directory with `rmdir()`
  - **If it's a directory and `-r` is NOT set:** prints an error message
- Tracks deletion counts and prints a summary table if `-s` is set

**Usage:**
```
custom_rm [OPTIONS] <file1> [file2 ...]
```
| Flag | Description |
|------|-------------|
| `-r` / `-R` | Remove directories and their contents recursively |
| `-v` | Verbose — print each file/directory as it is removed |
| `-s` | Print deletion summary (files and dirs removed) |

```bash
./bin/custom_rm temp.txt
./bin/custom_rm -rv old_project/
./bin/custom_rm -rs build/ dist/
```
